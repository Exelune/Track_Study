# Shader란?
오브젝트가 어떤 식으로 보일지 표현을 결정하는 프로그램이다.

GPU에서 실행되며, Vertex에서 형태를 계산하고 Fragment에서 픽셀 색을 계산한다.

<br>

# Shader Graph(URP, HDRP)
URP에서의 Shader Graph는 Lit(조명연산 O)과 Unlit(조명연산 X)으로 나눈다. 우선 Lit만 다룰 것이다.

Shader Graph는 코드를 작성하지 않고 셰이더를 만드는 기능이다.

노드기반 시스템이며 블럭이나 노드를 추가하여 기능을 더할 수 있다.(노드를 누르고 f1을 누르면 당 노드의 설명이 적혀있는 온라인 창을 띄워준다.)

<img width="559" height="757" alt="image" src="https://github.com/user-attachments/assets/6b028fdb-3904-4053-bf2f-2e5fb2fbce9f" />

### Vertex(형태/위치)
- Position (위치)
- Normal (방향)
- Tangent (방향 축)

### Fragment(색/질감 등등)
- Base Color (오브젝트의 기본 색)
- Normal (노말 맵)
- Metallic (금속성)
- Smoothness (반사 정도)
- Emission (자체 발광)
- Ambient Occlusion (그늘 보정)

<br>

<img width="266" height="549" alt="image" src="https://github.com/user-attachments/assets/a1e8df73-da28-4aa9-adb4-7b3f551ff785" />

그래픽 인스펙터에서는 2개의 창이 있다. Node Setting / Graph Setting 으로 나눠져 있다.

Node Setting은 해당 셰이더 그래프의 변수 인스펙터를 띄워준다.

Graph Setting은 이 셰이더 그래프의 설정을 건드릴 수 있다.

<br>

<img width="356" height="438" alt="image" src="https://github.com/user-attachments/assets/a455bfab-ed4e-4824-8277-9fbb856fc72c" />

여기서 여러 변수를 만들 수 있고, Exposed가 True로 설정되어 있으면 외부에서 값을 건들 수 있다.

<br>

## Shader Graph 장점
- 노드 기반 시스템이라서 제작하기가 쉽다.
- 실시간으로 Preview를 보면서 작업 가능
- 다양한 효과 구현이 가능하다.
- 코드와도 연동이 가능하다(HLSL)

<br>

## Shader Graph 단점
- 복잡한 셰이더를 만들수록 노드가 많아진다.(어려워지거나 복잡해짐)
- 결국 노드 기반은 코드 기반보다는 제한적이다.(유연성 등)
- 빌트인에서는 사용이 불가능하다.

<br>

## 결론
Shader Graph는 머티리얼의 표현을 노드 기반으로 만드는 기능이며, 여러 노드나 블럭을 추가하여 다양한 표현을 할수 있다.

하지만 복잡한 로직이나 최적화가 중요할 경우에는 코드 기반의 셰이더가 더 적합하다.

<br>

## 활용법
캐릭터나 사물에 발광 효과나 특수한 효과 같은 게 필요할때 사용할 수 있어 보인다.

또는 간단하게 셰이더가 필요할때도 만드는 용으로 사용할 수 있을 것 같다.
